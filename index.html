🗺️ The 5-Month Roadmap

Month 1: Arrays, Strings, Two Pointers, Sliding Window, Hashing (fundamentals)
Month 2: Binary Search, Linked Lists, Stacks, Queues, Trees
Month 3: Graphs (BFS/DFS), Heaps, Union Find, Tries
Month 4: Dynamic Programming (all variants), Backtracking, Greedy
Month 5: Full mock interviews, revision, system design basics, behavioral prep

🧠 Google's Interview Style

Google prioritizes thought process over final answer. Always think aloud. They test: Can you break down a problem? Handle edge cases? Optimize iteratively? Start with a brute force, clearly state its complexity, then improve. Interviewers often redirect or add constraints — be flexible.

⏱️ The 45-Minute Framework

0–5 min: Understand & clarify (ask about constraints, edge cases, input size)
5–10 min: Discuss approach aloud, brute force first
10–35 min: Code the optimal solution cleanly
35–40 min: Test with examples & edge cases
40–45 min: Complexity analysis & follow-ups

📊 Complexity Mastery is Non-Negotiable

Every solution must be followed by time and space complexity analysis — unprompted. Google interviewers expect you to know O(n log n) vs O(n²) implications. Practice saying "This is O(n) time and O(1) space because…" out loud after every problem you solve.

🔥 Google's Favorite Patterns

BFS/DFS on graphs and trees, Sliding Window, Two Pointers, Binary Search on answer, DP (particularly on strings/intervals), Topological Sort, and Dijkstra's. Master these patterns deeply — ~70% of Google problems map to these 8 patterns.

📝 Code Quality Matters

Write clean, readable, production-style code. Use meaningful variable names (not a, b, c). Modularize with helper functions. Avoid hacky tricks. Google engineers review your code as if it's going into production. Comments explaining non-obvious logic are appreciated.

💬 Master the Behavioral Round (Googleyness)

Prepare 8–10 STAR stories covering: leadership, conflict resolution, failing and learning, handling ambiguity, impact, and working cross-functionally. Google assesses Googleyness — intellectual humility, comfort with ambiguity, and bias to action. Practice with a partner weekly.

🏗️ System Design (For Senior Roles)

For L4+: Study designing URL shortener, YouTube, Google Search, Google Maps, Gmail. Focus on scalability, consistency, availability tradeoffs. Read "Designing Data-Intensive Applications" by Kleppmann. Practice drawing architecture diagrams and explaining component choices.

📅 Daily Practice Ritual

Phase 1 (Months 1–3): 3 new problems/day + 1 revision
Phase 2 (Month 4): 2 new + 2 revisions + 1 timed mock
Phase 3 (Month 5): Full mock interviews daily (use Pramp, Interviewing.io). Track weak patterns in a spreadsheet. Quality over quantity — understand the why deeply.

⚡ Are 350 Problems Enough?

Yes — with depth. These 350 are the highest-frequency Google interview problems. The goal isn't quantity but truly internalizing the patterns. If you understand why the solution works (not just how), and can solve unseen variations in 20–30 min, you are ready. Review each problem 2–3 times.
